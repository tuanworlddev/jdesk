# jdesk-codegen

Annotation processor (`dev.jdesk.codegen.DesktopCommandProcessor`) implementing
compile-time command registration (ADR-005, spec section 11). Registered both via
`META-INF/services/javax.annotation.processing.Processor` (annotation-processor path /
classpath discovery) and via `provides ... Processor` in `module-info.java` (module
path).

## Generated Java

For every public top-level class that declares `@DesktopCommand` methods, the processor
generates, in the same package:

- `<ServiceClass>Commands` — a final class with
  `public static CommandRegistry create(<ServiceClass> instance)` containing one
  `CommandDefinition` per command, sorted by wire name. Timeouts are always
  `Optional.empty()` (the runtime default applies; `@DesktopCommand` carries no timeout
  attribute).
- `JDeskCommands` — generated once per package **only when that package declares more
  than one service class**. It has
  `public static CommandRegistry combine(CommandRegistry... registries)` so an app can
  compose the per-service registries:

  ```java
  CommandRegistry registry = JDeskCommands.combine(
          GreetingServiceCommands.create(greetings),
          FileServiceCommands.create(files));
  ```

  Apps with services spread across packages can call any generated `JDeskCommands`
  aggregator (its content is package-independent) or build their own composition on top
  of `CommandRegistry.of(...)`.

## Command method shape

```java
public CompletionStage<Res> name(Req request, InvocationContext ctx)  // payload command
public CompletionStage<Res> name(InvocationContext ctx)               // no payload
public CompletionStage<Res> name()                                    // no payload, no context
```

`Req` must be a public record, `String`, or a boxed primitive. `Res` additionally allows
`Void`. Record components may be primitives, boxed primitives, `String`, other public
records, `List<X>`, `Map<String, X>`, and `Optional<X>`, recursively. Rejected at compile
time: duplicate or grammar-violating command names, missing/conflicting capability
annotations, non-public methods/classes/DTOs, non-record DTOs, `Object`, `Class`,
`Method`, `Throwable`, `MemorySegment`, wildcards, raw generics, arrays, generic records,
recursive records, overloaded command methods, name/namespace conflicts (`a.b` vs
`a.b.c`), and return types other than `CompletionStage<Res>`.

## Generated TypeScript

Emitted once per compilation at end of processing:

- default location: `CLASS_OUTPUT/jdesk-ts/types.ts` and `.../commands.ts`;
- override with `-Ajdesk.ts.outputDir=/path/to/dir` (files are then written to that
  directory instead).

`types.ts` holds one interface per DTO record (sorted by name; interface name is the
record's simple name — simple-name collisions across packages are a compile error).
Mapping: `String`/`char` → `string`, numeric primitives/boxes → `number`, `boolean` →
`boolean`, `List<X>` → `X[]`, `Map<String, X>` → `Record<string, X>`, `Optional<X>` →
`X | null`, records → interfaces, `Void` response → `void`.

`commands.ts` exports a nested `commands` object built from the dot-separated wire names
(`commands.greeting.greet(req)`), wired to `invoke(name, payload)` imported from
`@jdesk/client` (the static runtime in `js/jdesk-client`).

## Determinism

Identical input produces byte-identical output: no timestamps, no absolute paths, fixed
`\n` newlines, commands sorted by wire name, record components in declaration order,
interfaces and imports sorted by name. Every generated file starts with
`// Generated by jdesk-codegen. Do not edit.` Golden tests under
`src/test/resources/golden/` compile the same input twice and compare bytes.
