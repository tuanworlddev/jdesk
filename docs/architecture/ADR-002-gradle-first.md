# ADR-002: Gradle-first, Maven-compatible publication

Status: accepted (spec section 3, ADR-002)
Date: 2026-07-11

## Decision

Gradle multi-project build, Kotlin DSL, checked-in wrapper (Gradle 9.6.1, bootstrap
generated by a locally installed Gradle 9.6.1). Kotlin appears only in `build-logic/`
and `*.gradle.kts`; all production sources are Java.

- Java toolchains pin JDK 25 reproducibly (foojay resolver for CI provisioning).
- Convention plugins live in `build-logic/` (`jdesk.java-conventions`,
  `jdesk.library-conventions`).
- Publication: Maven artifacts + Gradle module metadata + sources + javadoc + BOM.
- Maven consumers can use runtime artifacts; a Maven *packaging plugin* is explicitly
  not part of v1 (documented truthfully in README/docs).
- Gradle TestKit executes real consumer builds for plugin verification.
- Dependency versions centralized in `gradle/libs.versions.toml`; dependency
  verification metadata and lockfiles are checked in.

## Non-JPMS exception

`jdesk-gradle-plugin` has no `module-info.java`: Gradle plugins load on Gradle's
non-modular runtime classpath. This is a build-logic artifact, not a production
runtime module, so the JPMS rule of ADR-001 does not apply to it.
